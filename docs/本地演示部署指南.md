# 本地演示部署指南

## 方案说明

**架构：**
- 后端：部署在云服务器（公网IP访问）
- 小程序：本地微信开发者工具演示

**优势：**
- ✅ 无需域名和备案
- ✅ 无需 HTTPS 证书
- ✅ 成本低（仅需云服务器）
- ✅ 适合毕设答辩演示

---

## 一、云服务器准备

### 1.1 购买云服务器

**推荐配置：**
- CPU：2核
- 内存：4G
- 系统：Ubuntu 20.04 / CentOS 7
- 带宽：1M 起步

**学生优惠平台：**
- 阿里云学生机：~10元/月
- 腾讯云学生机：~10元/月
- 华为云学生机：~9元/月

### 1.2 安装环境

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 JDK 8
sudo apt install openjdk-8-jdk -y
java -version

# 安装 MySQL 8.0
sudo apt install mysql-server -y
sudo mysql_secure_installation

# 安装 Redis
sudo apt install redis-server -y
sudo systemctl start redis
sudo systemctl enable redis

# 安装 Maven（可选，本地打包即可）
sudo apt install maven -y
```

### 1.3 配置安全组规则

**在云服务器控制台添加规则：**

| 端口 | 协议 | 来源 | 说明 |
|------|------|------|------|
| 8080 | TCP | 0.0.0.0/0 | Spring Boot 后端 |
| 3306 | TCP | 0.0.0.0/0 | MySQL（可选，建议仅本地） |
| 6379 | TCP | 127.0.0.1 | Redis（仅本地） |
| 22 | TCP | 0.0.0.0/0 | SSH |

## 三、后端部署

### 3.1 本地打包

```bash
# 进入后端目录
cd E:\GraduationProject\campus-delivery\server

# 修改 application.yml 配置
# 将数据库地址改为云服务器IP

# Maven 打包
mvn clean package -DskipTests

# 生成的 jar 包位于：target/delivery-1.0.0.jar
```

### 3.2 上传到服务器

```bash
# 使用 SCP 上传（替换为你的服务器IP）
scp target/delivery-1.0.0.jar root@你的服务器IP:/opt/campus-delivery/

# 或使用 FTP 工具（FileZilla、WinSCP）上传
```

### 3.3 启动后端服务

```bash
# SSH 登录服务器
ssh root@你的服务器IP

# 创建目录
mkdir -p /opt/campus-delivery
cd /opt/campus-delivery

# 后台运行
nohup java -jar delivery-1.0.0.jar > app.log 2>&1 &

# 查看日志
tail -f app.log

# 查看进程
ps aux | grep java

# 停止服务
pkill -f delivery-1.0.0.jar
```

### 3.4 测试后端接口

```bash
# 在本地电脑测试（替换为你的服务器IP）
curl http://你的服务器IP:8080/api/merchant/list

# 应该返回 JSON 数据
```

## 六、常见问题

### 6.1 小程序无法连接后端

**检查清单：**
- [ ] 云服务器 8080 端口是否开放
- [ ] 后端服务是否正常运行（`ps aux | grep java`）
- [ ] IP 地址是否正确
- [ ] 微信开发者工具是否关闭域名校验

**测试命令：**
```bash
# 在本地电脑测试
curl http://你的服务器IP:8080/api/merchant/list
```

### 6.2 数据库连接失败

**检查 application.yml：**
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/campus_delivery
    username: root
    password: 你的数据库密码
```

### 6.3 跨域问题

已配置 `CorsConfig`，允许所有来源访问。如仍有问题，检查：
```java
// CorsConfig.java
.allowedOriginPatterns("*")  // 允许所有来源
```

### 6.4 服务器内存不足

```bash
# 限制 JVM 内存
java -Xmx512m -Xms256m -jar delivery-1.0.0.jar
```
